# AI 味与查重测试工具区

本目录用于集中管理 `chinese-official-writing` 的 AI 味审查、查重和同质化测试入口。实际样稿、依赖、模型缓存和运行输出放在 `output/ai-dedupe-roundN/`，不提交到 git。

## 工具优先级

1. **Agent skill 优先**：优先用独立 subagent 加载 AI 味审查、去重、查重或公文审稿类 skill 进行测试。只给“原任务 + 样稿”，不提供主线程结论，不让测试样稿外发到不必要的第三方 API。
2. **不用 API 的开源工具第二优先**：优先使用本地可运行工具，例如仓库自带 `prose_lint.py`、`jieba`、`simhash`、TF-IDF/cosine、`text2vec` 本地模型、Hello-SimpleAI 中文检测模型的本地权重等。
3. **需要 API 的工具最后考虑**：腾讯云 Text AIGC Detection、商业查重或平台 API 只在明确需要时使用；使用前先脱敏、记录接口、样本范围、是否外发文本和失败回滚口径。

## Round 约束

- 所有测试以 GitHub `origin/main` 最新 commit 为基线。
- 每轮至少由 subagent 生成或复核样稿，主线程不自行充当唯一裁判。
- 只记录三轮以上重复出现的共性问题作为修复候选；单轮三份样稿都出现的问题只作为观察项，不能直接触发修复。
- 每次修复必须与上一轮基线做消融测试；消融不通过则回滚该次修改。
- 不复制社区 skill 的代码、正则、模板正文、大段 prompt 或固定话术，只抽象流程和检查维度。

## 候选工具池

已核验的候选指标记录在根目录 `agent.md`。后续新增工具时，把工具放入本目录下的子目录或说明文件，并记录：

- 来源链接和版本；
- 是否需要 API；
- 是否支持中文；
- stars、downloads、更新时间或平台下载量；
- 实际运行命令和输出路径；
- 样稿是否离开本地环境。
