# Model-Agnostic A/B/C Ablation Summary

本文件只记录脱敏评估结论，不包含用户材料、内部文件内容、个人信息或原始公开公文全文。

## 测试设计

- 覆盖文体：27 类。
- 每类任务：10 次。
- 总任务数：270 个。
- Writer A：当前 agent 或指定模型在已安装 `chinese-official-writing` Skill 的条件下生成样稿。
- Writer B：当前 agent 或指定模型不读取 Skill，仅按普通提示生成样稿。
- Evaluator C：独立上下文仅根据 A/B 输出和公开公文风格参照进行评估。
- 公开原稿处理：发布包不保存原文，只保存脱敏后的风格参照和聚合评估结论。

## 运行结果

- 评估批次数：9。
- A/B 写稿有效批次：9/9。
- C 独立评估有效批次：9/9，其中 2 个批次为补跑评估。
- 全量运行耗时：约 3519 秒。
- 原有效评估批次可自动解析出若干明确胜负表述；由于各批次评估粒度不完全一致，不折算为 270 个任务逐条胜率。补跑批次覆盖通知、请示、报告、工作要点、工作总结、调研报告，均形成 Writer A 胜出结论。补跑只针对同一批 A/B 样稿重新调用 Evaluator C，未重写 A/B 样稿。

## 补跑批次

- batch-01: 原 Evaluator C 空返回，补跑评估形成通知、请示、报告均为 Writer A 胜出的结论。
- batch-06: 原 Evaluator C 空返回，补跑评估形成工作要点、工作总结、调研报告均为 Writer A 胜出的结论。

## 原始输出位置

- 原始 A/B/C 输出保存在 `output/agent-public-ablation`，该目录被 `.gitignore` 排除，不进入发布包。

## 复核结论

Evaluator C 的独立评估用于观察技能规则是否改善文种适配、视角控制、公文语气、反 AI 句式和算力类论证链条。该测试不能替代人工审稿，事实、政策依据、金额和采购结论仍需业务审核。

## 结论说明

本轮全量生成完成 270 个任务，Writer A 与 Writer B 均完成对应样稿，共形成 540 段对比材料。Evaluator C 按 9 个批次出具文种级评估意见；其中 2 个批次曾空返回，已在不重写 A/B 样稿的情况下补跑 C 轮。补跑覆盖通知、请示、报告、工作要点、工作总结、调研报告 6 个文种，均形成 Writer A 胜出结论。上述信息用于回归测试和相对风格对比，不作为逐任务胜率发布。
