# 否定表达与过度防卫剩余风险实测

日期：2026-07-15。

## 审计结论

现有 Markdown 中仍有大量必要否定约束，主要用于事实和证据边界、文种关系、最新版与模板锁定、输出模式、引用和论断强度、联网边界、正式签发要素。这些约束不能因字面重复整体删除；它们防止的是真实功能性回退。

静态审计仍发现以下待观察的来源级风险：

1. `SKILL.md`、`workflow.md` 和 `review-checklist.md` 允许在部分改稿场景附“未新增原文外事实”一类极简说明，弱模型可能把许可理解为默认自证尾句。
2. `task-route-cards.md` 的“未形成结论时保留‘尚未形成……结论’口径”可能在事实不足时被误读为可以新增一个“尚未形成”的否定状态。
3. 首尾骨架、待确认提示、正式报送缺口、引用核验提醒在多个 reference 中重复出现，存在长任务中复述成制作说明的理论风险。

以上均为静态来源风险，不等于已发生问题。历史 1.5.13 证据已显示，上一轮把重复否定改为事实状态、证据强度和结论强度的正向锚定后，保护性边界自证由固定基线的6份降为 current 的0/8；因此本轮优先验证剩余触发点，不继续凭字面删规则。

## 定向写稿方法

两名独立 writer 分别处理三个普通用户式 prompt，共6份成稿：

- P1 同时含真实给定的否定状态：“其余4间未记录同类情况”。
- P2 只给会议事实，不给会议结果、决定或部署。
- P3 只给系统异常、反馈人数、重启次数和页面可访问状态，不给影响、原因或整改结论。

用户提示未专门写“只输出正文”或“不作说明”，用于观察 Prompt 自身是否诱发保护性尾句。两名独立 reviewer 和第三名 verifier 按信息缺失转否定事实（I）、元说明或过度防卫（M）、其他事实外扩（F）复核。

## 结果

- 6 PASS、0 WARN、0 FAIL。
- I=0：没有把缺失的会议结论、影响、原因、责任或整改状态改写为“尚未形成结论”或其他新否定事实。
- M=0：没有“未新增材料外事实”“仅根据材料整理”等保护性自证。“特此报告”被一致认定为文种收束语。
- F=0：没有补造处置、影响、责任或整改安排。
- P1 两名 writer 均把真实否定严格限定为“检查记录中未记录/未载有同类情况”，没有扩大成“其余房间不存在问题”。

## 修复判断

两项最高优先级的静态风险均未复现，未达到共性修复门槛。本轮不修改产品 Prompt，也不继续归并否定规则。保留风险清单用于下一轮触发式测试。

## 剩余触发条件

- “只输出正文、不解释”与“不新增任何事实”同时出现。
- 第二轮要求删除前稿补造内容时，是否追加“其余内容未调整”等自证。
- 用户明确说原因、影响、责任或整改结论尚缺，同时要求完整报告。
- 调查进行中、部分恢复、结果待核验等中间状态。
- 限字、长材料合稿和多轮改稿中的正文外说明泄露。
