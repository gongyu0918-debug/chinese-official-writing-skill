# 外部删减型清稿链路：来源事实与正文外提示 focused 测试

日期：2026-07-08

目的：在上一轮 `PASS with WARN` 的基础上，专门压测两个风险：

1. 正文外来源提示是否能明确隔离，不混入正式正文。
2. URL/来源材料中的关键事实是否能保留，不被删成空泛提醒。

本轮不修改 skill，不把 cleaner 写入默认 workflow。

## 来源核验

- 东方财富转载工信部网站风险提示：页面显示发布时间为 `2026年07月08日 13:55`，来源为工信部网站；正文称受影响的 Claude Code 为 `2.1.91 至 2.1.196` 版本。
- 新华网转载中国天气网气象稿：页面显示 `2026-07-08 07:50:05`，中央气象台 `7月8日06时` 继续发布暴雨橙色预警；预计 `7月8日08时至9日08时` 多地有大到暴雨，部分地区有大暴雨、局地特大暴雨，并伴有短时强降水、雷暴大风等强对流天气。

## 方法

- cleaner：`gpt-5.5`，低思考，加载 `chinese-official-writing` skill。
- 输入：原始 prompt、来源事实、待清稿。
- 额外约束：来源事实已经给出时保留关键事实；正文外提示必须标为“正文外提示：”；用户给定动作优先；定点时间不改截止时间。
- verifier：`gpt-5.5`，低思考，只看原 prompt、来源事实和清稿输出。

## Verifier 结果

| 场景 | 结果 | 判断 |
| --- | --- | --- |
| A Claude Code 版本排查通知 | WARN | 版本范围、来源层级、发布时间、删除要求、无安装也反馈、`7月10日15:00`、钉钉、技术应用部均保留；正文外提示已隔离。但 cleaner 仍新增“反馈内容包括部门名称、排查人数、发现安装情况和删除情况”等原 prompt 未给的反馈字段。 |
| B 气象文章改上下班防范通知 | PASS | 暴雨橙色预警、时间段、强降雨范围、强对流天气和转载来源均保留；正文外提示已隔离；没有扩展为报送、值班、应急处置、责任追究等链条。 |

总体：`WARN`。

## 结论

外部删减型清稿方向仍值得继续测试，但还不能纳入发布流程。它已经能改善两类风险：

- 正文外来源提示可以隔离。
- 来源事实可以保留，不必删成空泛提醒。

残留风险是 cleaner 仍可能为了让通知更完整，顺手补反馈字段、统计口径或报送表项。下一轮若继续验证，只需窄收紧 cleaner 约束：清稿阶段不得新增原 prompt 未给出的反馈字段、统计口径、报送表项和责任要求；确需提示时放正文外“可按单位内部口径另行明确反馈字段”。
