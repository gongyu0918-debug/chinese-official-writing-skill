# 3000 字以上长文真实回归测试

日期：2026-07-06

本轮补测 3000 字以上调研、可研和多附件合稿场景。测试目标是确认当前 `chinese-official-writing` 在长程写作中仍能保持文种结构、事实保真、附件合并完整性和缺项边界，不因新增 proofreading workflow 或既有长文规则导致功能性回退。

本轮为只读真实写稿测试，不修改 skill prompt/reference。独立 verifier 未发现多数样本共性失败，因此不做本轮 prompt 修复。

## 测试方式

- Writer R3000：`019f36c5-6df9-7373-be07-b8db09381ca6`
- Writer F3000：`019f36c6-bf50-78c3-be98-954409ea308b`
- Writer M3000：`019f36c6-d354-7482-a9f0-e2ac7af61ae4`
- Verifier：`019f36cd-6d88-73c3-9342-fe75eb03400a`

Writer subagents 均加载 `.agents/skills/chinese-official-writing/SKILL.md`，只输出正式稿正文，不修改仓库。Verifier 只看“原测试要求 + 三份样稿”，不加载写作 skill，不复写正文，按 PASS/WARN/FAIL 复核。

## 覆盖用例

### R3000 调研报告

题目：《校园学习空间使用情况调研报告》

材料约束：

- 调研单位：图书信息中心。
- 时间：2026 年 5 月至 6 月。
- 方法：线上问卷、现场走访、座谈访谈、系统预约数据梳理。
- 样本和事实：486 份有效问卷；主馆晚间座位使用率高峰约 92%；考试周部分区域超过 95%；普通周白天部分学院公共学习区使用率不足 45%；共享研讨间预约成功率约 61%；38% 受访学生认为插座数量不足；32% 反映夜间返回宿舍动线不够便利；27% 希望增加静音区。
- 禁止补造预算、联系人、空间点位清单、实施批次等未给事实。

结果：

- Writer 输出明显超过 3000 字，结构包含调研背景、调研方式、基本情况、主要问题、原因分析、对策建议和结尾。
- 关键数字和调研方式均进入正文。
- 预算、联系人、空间清单、设施建设批次等未给事实进入“补充以下信息后，报告会更完整”。
- Verifier 判定：PASS。

### F3000 可行性研究报告

题目：《校园统一预约服务平台建设可行性研究报告》

材料约束：

- 现状：图书馆座位预约系统已运行，月均预约约 4200 人次；体育场馆由管理员登记；报告厅主要通过 Excel 台账和电话确认；学生活动室由二级学院线下审批。
- 问题：预约入口分散、资源状态不透明、时间冲突发现滞后、取消和变更通知不及时、统计口径不一致、跨部门协调成本高。
- 目标和功能：统一资源目录、统一申请入口、统一审批流、统一消息通知、统一统计分析、冲突预警；统一账号登录、资源分类检索、在线预约申请、审批流配置、变更取消、短信或站内信通知、管理员排期、数据统计、异常冲突提示、操作留痕。
- 路径：先试点图书馆研讨间、报告厅、学生活动室，再逐步纳入体育场馆和其他公共资源。
- 禁止补造预算、供应商、具体上线日期、网络安全等级保护结论、联系人、验收指标细化表。

结果：

- Writer 输出明显超过 3000 字，结构包含项目概况、必要性、现状和问题、建设目标、建设内容、实施路径、可行性分析、风险控制、预期成效和结论。
- 月均约 4200 人次、Excel/电话确认、二级学院线下审批、试点路径等关键事实均保留。
- 预算、供应商、开发周期、安全等级、联系人和验收指标列为待确认或后续专项审查事项，没有被写成确定事实。
- Verifier 判定：PASS。

### M3000 多附件合稿

题目：《2026 年上半年校园服务保障情况报告》

附件约束：

- 附件一后勤：维修工单 1286 件、按期办结率 94.6%；食堂检查 18 次、整改提醒 6 项；消防安全演练 2 次；未解决问题包括老旧管网渗漏反复、暑期集中维修窗口较短、个别报修点位重复返修。
- 附件二图书与信息：图书馆座位预约 312000 人次、高峰时段阅览区使用率接近 95%；新增自习座位 120 个；试点线上知识库答疑；学生反映电源插座不足、噪声管理不稳定、研讨间预约规则理解不一致；信息服务部门完成校园网故障处置和业务系统账号支持若干次，但未给具体次数。
- 附件三学生事务：宿舍安全检查 7 轮，发现违规用电、私拉插排等问题 43 项；心理健康宣传和团体辅导活动 12 场，参与学生 1860 人次；配合晚自习、社团活动、宿舍管理现场协调；部分活动场地预约与晚自习安排衔接不够顺畅。
- 禁止补造“未发生安全事故”、满意度、投入经费、人员名单、领导讲话、新增制度名称等未给事实。

结果：

- Writer 输出达到 3000 字以上，覆盖三个附件全部条线。
- 1286 件、94.6%、18 次、6 项、2 次、312000 人次、95%、120 个、7 轮、43 项、12 场、1860 人次等关键数字均保留。
- 对“信息服务若干次”未扩写成具体次数。
- 未出现“未发生安全事故”、满意度、经费、领导讲话、制度名等编造结论。
- 待确认事项覆盖未给具体次数、点位清单、复核情况、预算、责任分工和期限。
- Verifier 判定：PASS。

## 独立 verifier 结论

总体结论：PASS。

- R3000：PASS。篇幅明显超过 3000 字；文种符合调研报告；保留 2026 年 5-6 月、486 份问卷、92%、95%、45%、61%、38%、32%、27% 等关键事实；未编造预算、联系人、具体空间清单；后半程建议较充分。
- F3000：PASS。篇幅明显超过 3000 字；文种符合可研报告；保留图书馆座位预约月均约 4200 人次、体育场馆管理员登记、报告厅 Excel/电话确认、学生活动室二级学院线下审批和试点路径；未编造预算、供应商、上线日期、安全等级、联系人。
- M3000：PASS。篇幅达到 3000 字以上；完整覆盖三份附件；保留全部关键数字；未把“若干次”扩写为具体次数；未编造事故结论、满意度、经费、领导讲话或制度名。

共性问题：未形成共性阻断问题。

是否建议做 prompt 修复：否。三份均达到长文、文种、事实保留和不编造的核心要求，未出现多数样本共同失败。个别敏感项均以待确认或后续审查方式处理，未构成事实编造。

## 本轮处理结论

接受结论：

- 当前 1.5.1 后续本地候选在 3000 字以上调研/可研/多附件合稿中，没有发现功能性回退。
- 长文后半程没有明显头重脚轻，措施和待确认事项能够延续到正文末尾。
- 多附件合稿没有遗漏附件，也没有把未给事实写成确定结论。

拒绝扩大修复：

- 不因本轮单次长文测试新增 prompt 约束。
- 不把“待确认事项”扩展成更强审查流程。
- 不引入脚本清洗、联网核验、硬门禁或新模型。

剩余风险：

- 本轮只覆盖 3 个 3000 字以上长文样本，不能代表所有长文场景。
- 未覆盖 5000 字以上超长报告、带真实附件文件的合稿、跨多轮连续改稿后的长文重排。
- 未做外部事实核验；本 skill 仍只做稿内事实保真和风险提示。
- 未验证 Word/docx 排版交付，结论只针对流式文本输出。
