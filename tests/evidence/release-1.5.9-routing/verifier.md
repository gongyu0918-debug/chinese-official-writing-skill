# 1.5.9 发布前真实写稿独立 verifier

## 审核范围与口径

- 仅审核 A—F 的原 prompt 与直接成稿。
- 判定项：指令遵循、路由/文种结构、给定事实是否进入、是否编造事实/数据/文献、证据状态、是否残留过程说明、论文构造建议是否与正文分开。
- `PASS` 表示未发现影响交付的问题；`WARN` 表示存在轻微但非阻断风险；`FAIL` 表示违反用户明确要求、改变事实或导致交付不可用。
- 证据文件自身的 Markdown 标题不计入成稿格式；正式公文直接成稿中的 Markdown 才作为格式风险检查。

### A 公文起草

- 通知标题、主送对象和正文安排符合“举办培训并报送名单”的任务性质。
- 时间、地点、人数、截止时间、邮箱、联系人和电话全部进入正文，未改变数值与对象。
- 未补写发文机关、落款、日期、培训内容或其他未给事项，也没有过程说明或 Markdown 正文标记。

### B 普通课程论文提纲

- 输出为题目下三级提纲，覆盖研究问题、问卷设计、实施计划、数据分析方案、结论与不足，符合课程论文提纲任务。
- “问卷实施计划”“问卷回收后的数据清理”等均以计划或后续任务表述，没有把尚未发放的问卷写成已实施，也没有生成结果、样本数据或文献。
- “可补充材料”“可补充论点”“可展开论点”位于提纲之后，和拟交付提纲清楚分开；建议均使用条件式表达，没有预设调查结论。

### C 开题报告

- 采用研究对象、研究问题、研究内容、研究方法与实施安排、预期产出的开题结构，未误写为已完成论文。
- “拟采用”“尚未发放”“尚未形成”“尚需确定”“不预设”等表述准确保留本科开题和研究设想状态。
- 未补问卷结果、样本数量、关系方向、因果结论或具体文献；正文后的补充材料和论点建议单列，未混入已知事实。

### D 独立文献综述

- 三条给定材料均进入综述，作者、年份及各自观点没有错配。
- 张某材料保持为“认为”，李某材料保持“部分学生报告”的范围，王某材料保持“可能关联”；没有把观点、报告或讨论升级为普遍事实、影响、效果或因果。
- 对三项材料关系和证据缺口的归纳有给定材料支撑，未补论文题名、方法、样本、数据或新文献。
- 补充材料和可展开论点在综述正文后单列，且继续使用条件式、待核验表述。

### E 公文改稿

- 保留标题和唯一可确认的巡检事实，删除了“全面消除风险”的结论以及信息中心、每周督办、长效闭环等未获事实支持的内容。
- 未以新的整改状态、责任主体或处置安排填补删除后的空白；输出只有改后正文，无过程说明和 Markdown 正文标记。

### F 论文改稿

- 准确写明研究仍处设想阶段、问卷尚未发放，并据此撤回“证明”“必然导致”“已经验证”等无证据结论。
- “现阶段无法判断”没有反向断言不存在相关或因果关系，也未补数据、方法结果或文献。
- 输出仅为改后正文，没有审核旁白、过程说明或额外建议。

## 总体结论

A—F 均为 `PASS`，未发现发布阻断项。六份成稿均遵循各自任务边界；公文起草与改稿没有论文结构混入，论文提纲、开题报告、独立文献综述和论文改稿也没有公文结构混入。给定事实均得到保留，未发现编造数据、事实或文献，研究计划、主观报告、相关关系与因果结论的证据状态保持准确。需要构造建议的论文样本均将建议置于正文之后并明确分隔。
