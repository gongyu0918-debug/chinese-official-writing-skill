## 检查 Prompt 与输出结果

### R01 食品安全专项检查情况报告

Prompt：

> 请给市市场监管局写一篇食品安全专项检查情况报告，约900字，必须是报告不是请示，不要写“妥否，请批示”，要包含检查范围、发现问题、整改措施、下一步计划。材料：本轮检查覆盖校园周边餐饮、农贸市场、网络订餐商户三类场景；共抽查点位36个，发现台账记录不完整、散装食品标签不规范、个别商户后厨分区标识不清三类问题；已现场责令整改，并拟组织复查。

输出结果摘要：

- 读取 `SKILL.md`、`genre-routing.md`、`handling-elements.md`、`argument-chains.md`、`final-review-layers.md`。
- 正文以“市市场监管局：”起笔，按“一、检查基本情况”“二、发现的主要问题”“三、整改处置情况”“四、下一步工作计划”展开。
- 已置入三类检查场景、36 个点位、三类问题、现场责令整改和复查安排。
- 结尾使用“特此报告。”，未写“妥否，请批示”。
- 自检判断文种正确、公文格式基本遵循、AI 味较低、prompt 要求落实。

Verifier 结论：

- 文种判断通过，未写成请示。
- 内容层面合格；格式轻微警告：作为完整正式报告，未输出发文单位落款和日期。

### R02 老旧小区电梯加装工作汇报

Prompt：

> 把下面基层材料整理成一篇约1200字的工作汇报，要求写完整，不要因为材料里有缺项就只问问题或草草结束；不要联网；语言要实在，少空话套话。材料：今年我们推进老旧小区电梯加装协调，已完成3个小区意见征集，2个小区进入方案设计，街道牵头召开协调会6次。难点是业主意见不一致、施工空间受限、资金分摊争议较多。我们做了三件事：一是建立居民代表沟通群，二是请设计单位现场踏勘，三是把费用分摊方案做成明白纸。缺少：最终审批依据、具体联系人、下一步时间节点。

输出结果摘要：

- 读取 `SKILL.md`、`handling-elements.md`、`argument-chains.md`、`official-style.md`、`final-review-layers.md`。
- 输出完整工作汇报，覆盖背景、进展、三类难点、三项工作、阶段效果、下一步安排和结尾。
- 没有联网，没有因缺最终审批依据、联系人、时间节点而中止写作。
- 缺项列在正文后的“待确认事项”：最终审批依据、具体联系人及联系方式、下一步明确时间节点。
- 自检判断完整成稿、缺项正文外、空话较少、没有头重脚轻。

Verifier 结论：

- Prompt 遵循通过，缺项没有变成阻断。
- 长文完整度通过，后半部分有阶段效果和下一步安排。
- 警告：标题残留 Markdown `**...**`；约 1200 字要求可能偏长，需字数工具确认；结尾有轻微套话但前文有事实支撑。

### R03 请示文本只审不改

Prompt：

> 只做审稿，不要重写全文。请检查下面这段是否有公文格式、文种判断、AI味和空话问题，并给出“位置 + 风险层级 + 修改建议”：
>
> 关于推进智慧社区建设的请示
> 区政府：
> 为全面赋能基层治理，打造数字化治理新生态，我街道拟上线智慧社区系统。系统将全面提升社区服务能力，形成长效闭环，推动治理能力现代化。项目预计需要一定资金支持，后续将按程序推进。特此报告。
> 街道办事处
> 2026年6月22日

输出结果摘要：

- 读取 `SKILL.md`、`genre-routing.md`、`review-checklist.md`、`anti-ai-patterns.md`、`final-review-layers.md`。
- 按位置逐条指出标题、主送、空话句、资金支持、结尾语、落款日期问题。
- 未重写全文；识别“请示”与“特此报告”文种收束错位。
- 指出“全面赋能”“新生态”“长效闭环”“一定资金支持”等空话与事实缺口。

Verifier 结论：

- 只审不改通过，文种错位识别通过，空话和事实缺口识别通过。
- 因 verifier 只拿到摘要，不能核验每条是否都严格包含“位置 + 风险层级 + 修改建议”；从摘要看方向正确。
