# 三稿匿名盲审包

三份稿件来自同一写作模型、相同 thinking、相同原始任务和相同宿主外壳下的独立上下文。来源标签已匿名化；只依据原始任务和成稿判断，不猜测来源，也不调用或读取任何公文写作 Skill。

## 裁决顺序

1. 先逐稿核对硬边界：事实与数字、主体与对象、未决状态和结论强度、文种与行文关系、用户格式或模板、Prompt 动作、输出模式。每项给 PASS/WARN/FAIL，并引用具体句子。
2. 用户明示篇幅范围时，篇幅只作为 Prompt 遵循项单列。两稿都在范围内时，长短差异退出质量比较；更接近区间中点不加分。两稿共同不合规时，不因某稿“更接近”而判优。篇幅合规不能单独构成胜出。
3. 再比较非篇幅质量：结构与信息利用、首中尾均衡、主体和宾语锚定、段落推进、可读性、自然度、机械句式和直接使用成本。信息写得更多本身不加分；材料外扩写不算信息利用。
4. “先……再……”“一是……二是……”或正式连接词单次出现不算机械。只有成簇重复、没有真实关系的连接词链、每段同构、边界自证循环、固定三段式、口号式收束或主体宾语反复缺失时，才判机械，并至少引用两处证据。
5. 直接使用成本分为：G0 可直接采用；G1 只需不超过 3 处局部修正；G2 需要重组一段以上、补结构要素或系统删重；G3 事实、文种、格式或输出模式失真，需要重写。范围内的篇幅差异不改变成本等级。
6. 给出三稿排序。第一名相对第二名只可判“严格胜出”“小胜”或“难分”。“严格胜出”要求硬边界不低、至少两个非篇幅质量优势、其中至少一项来自可读性/自然度/直接使用成本，并且没有实质质量维度更差。
7. 若一稿事实更稳而另一稿更自然，或裁决只能依赖篇幅、偏好或单句差异，判为难分，不强行分胜负。

## 输出格式

- 硬边界表：A/B/C 的逐项 PASS/WARN/FAIL 与原文证据。
- 质量比较表：各维度谁更好或相当、文本证据、采用影响。
- 排序：第一、第二、第三；第一相对第二为严格胜出/小胜/难分。
- 风险：是否存在任何一稿独有的事实、状态、文种、格式、Prompt 或输出模式硬回退。

### 稿件 A

数据中心机房制冷改造专题会议纪要

2026年7月9日14时30分，周明在第二会议室主持召开数据中心机房制冷改造专题会。办公室、信息中心、财务处、后勤处参加会议。会议听取了有关情况汇报，就机房运行情况、现有设备状态、前期维护情况、改造测算及后续研究事项进行了研究。现纪要如下。

会议听取了信息中心关于机房高温告警和现有制冷设备运行情况的报告。5月1日至6月30日，机房共出现高温告警9次，其中7次发生在13时至17时；期间记录的最高温度为31.8℃。现有3台精密空调中，1台已连续运行满8年，另外2台已连续运行满5年。与会单位围绕上述运行数据和设备使用年限进行了讨论。

会议听取了后勤处关于近期维护情况的报告。后勤处于6月20日完成一次清洗和冷媒检查，6月21日至30日期间机房仍出现高温告警2次。经讨论，目前可以确认清洗后告警次数有所减少，但现有情况尚不足以据此确定长期效果。有关后续判断仍需结合负荷测算和持续运行情况研究。

会议听取了财务处关于本年度设备更新预算和供应商初步报价情况的报告。本年度设备更新预算尚余48万元。两家供应商初步报价分别为45.6万元和52.3万元。因两家供应商报价范围不同，现阶段尚不能直接进行比较。对报价范围、改造内容及相关条件，需在后续研究中进一步核实。

办公室提出，在形成具体方案前，应先补齐负荷测算、施工窗口和业务连续性安排等基础情况，再提交馆长办公会研究。会议认为，负荷测算是后续研究改造事项的重要基础，应先行完成相关数据整理和测算工作。

会议一致同意，7月18日前完成负荷测算。信息中心和后勤处共同提供基础数据。有关单位应围绕机房现有运行数据、设备情况及维护情况做好基础资料提供工作，为后续研究提供依据。

会议对采用整体更换还是分步改造、是否在本年度实施、选择哪家供应商等事项均未作决定。施工窗口、停机安排、采购方式和验收指标尚未确定。上述事项待负荷测算等基础情况补齐后，再行研究。

### 稿件 C

**数据中心机房制冷改造专题会议纪要**

2026年7月9日14时30分，周明同志在第二会议室主持召开数据中心机房制冷改造专题会。办公室、信息中心、财务处、后勤处有关负责同志参加会议。会议听取了机房运行状况、设备维护情况、经费保障及前期准备工作汇报，围绕制冷改造必要性、实施条件和下一步工作安排进行了研究。现纪要如下。

会议指出，数据中心机房制冷系统运行年限较长，夏季高温时段保障压力较大。信息中心报告，5月1日至6月30日，机房共出现高温告警9次，其中7次集中发生在13时至17时，最高温度达到31.8℃。现有3台精密空调中，1台已连续运行8年，另外2台已连续运行5年。各单位要高度重视机房环境稳定对业务系统连续运行和数据安全的影响，持续加强运行监测、告警处置和日常巡检，防止高温风险扩大。

会议认为，设备维护能够在一定程度上缓解现阶段运行压力，但尚不足以作为判断长期保障能力的依据。后勤处报告，6月20日已完成一次清洗和冷媒检查，6月21日至30日机房仍出现2次高温告警。目前只能确认清洗后告警次数有所减少，尚不能据此确定长期效果。后勤处要会同信息中心继续跟踪维护后的运行数据，准确分析告警成因、设备能力和机房实际负荷，为后续决策提供可靠依据。

会议听取了经费和市场询价情况。财务处报告，本年度设备更新预算尚余48万元。两家供应商初步报价分别为45.6万元和52.3万元，因报价范围不同，尚不具备直接比较条件。有关单位要进一步梳理设备配置、辅材、施工、维保、税费及其他费用构成，统一询价口径，明确报价边界和服务内容，形成可比、可核的经费测算材料。

会议要求，在形成成熟方案前，不对整体更换还是分步改造、是否于本年度实施以及供应商选择作出决定。办公室提出的负荷测算、施工窗口和业务连续性安排等前置工作，应同步推进、相互衔接。信息中心和后勤处要于7月18日前共同完成机房负荷测算，分别提供设备运行参数、告警记录、环境数据、维护情况及相关基础资料，提出制冷能力评估意见和初步改造需求。

会议明确，施工窗口、停机安排、采购方式和验收指标尚未确定。信息中心要结合业务系统运行特点，研究施工期间的业务保障、应急处置和风险控制措施；后勤处要核实施工条件及配套需求；财务处要做好预算审核和资金保障准备；办公室负责统筹材料汇集、议题衔接和报送工作。待负荷测算、方案比选和业务连续性安排等材料补齐后，按程序提交馆长办公会研究。
