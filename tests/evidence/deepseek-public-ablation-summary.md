# DeepSeek A/B/C Ablation Summary

本文件只记录脱敏评估结论，不包含用户材料、内部文件内容、个人信息或原始公开公文全文。

## 测试设计

- 覆盖文体：27 类。
- 每类任务：10 次。
- 总任务数：270 个。
- Writer A：DeepSeek 在已安装 `chinese-official-writing` Skill 的条件下生成样稿。
- Writer B：DeepSeek 不读取 Skill，仅按普通提示生成样稿。
- Evaluator C：独立 DeepSeek 上下文，仅根据 A/B 输出和公开公文风格参照进行评估。
- 公开原稿处理：发布包不保存原文，只保存脱敏后的风格参照和聚合评估结论。

## 运行结果

- 评估批次数：9。
- A/B 写稿有效批次：9/9。
- C 独立评估有效批次：7/9。
- 全量运行耗时：约 3519 秒。
- 自动解析胜出计数：A=29，B=3，Tie=1。该计数仅从评估文本中抽取，最终以人工复核为准。

## 无效或待复跑批次

- batch-01: Evaluator C returned empty output
- batch-06: Evaluator C returned empty output

## 原始输出位置

- 原始 A/B/C 输出保存在 `output/deepseek-public-ablation-v2`，该目录被 `.gitignore` 排除，不进入发布包。

## 复核结论

DeepSeek C 的独立评估用于验证 Skill 是否改善文种适配、视角控制、公文语气、反 AI 句式和算力类论证链条。该测试不能替代人工审稿，也不代表具体事实、政策依据、金额或采购结论已经通过业务审核。

## 结论说明

本轮全量生成完成 270 个任务，Writer A 与 Writer B 均完成对应样稿，共形成 540 段对比材料。Evaluator C 对其中 7 个批次形成有效评估，覆盖 21 类文体、210 个任务；自动解析出的明确判断为 A=29、B=3、Tie=1，该数字只代表评估文本中可抽取的明确胜负表述，不等同于 210 个任务逐条人工判胜。有效评估中，Writer A 在多数批次中被判定更贴近公开中文公文风格，主要集中在方案、申请、函、复函、批复、公示、通报、会议纪要、可研报告、实施方案、建设方案、审查材料和 AI 算力技术文档。通知/请示/报告、工作要点/工作总结/调研报告两个批次因 C 空返回暂不纳入结论。
