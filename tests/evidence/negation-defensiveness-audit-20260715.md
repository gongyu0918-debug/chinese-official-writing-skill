# Markdown 否定表达与过度防卫轻量审计

日期：2026-07-15。

审计范围：canonical `chinese-official-writing/SKILL.md` 及其15个直接 reference；不把测试夹具、历史证据和平台镜像重复计为产品风险。

## 必须保留的约束类别

- 事实、证据、数据、引用、术语、否定范围和论断强度。
- 文种功能、行文关系和办理要素。
- 最新版底稿、用户模板、字段顺序和结构单元。
- 起草、改稿、复核、排版交付及输出模式。
- ANTI-AI 只作语义复核，不机械清洗、不改变事实。
- 默认联网边界及正式签发、Word 交付边界。

这些规则即使使用“不得、不补、不改写”等否定形式，也是在约束模型的常见越界行为。没有行为证据时直接删减，可能带来事实外扩、文种错位、旧稿回流或交付说明泄露。

## 有重复但暂未证明有害的类别

- 稀疏事实和缺项处理。
- 成品中的制作过程、内部受众、审核门禁、Markdown 残留。
- 最新版、模板、结构、首尾骨架。
- ANTI-AI 软审校和默认联网。
- 正式要素、占位符和文种边界。

这些内容存在跨文件重述，但不同 reference 服务于不同路由。只有真实加载链路证明重复导致正文应答式自证、突然否定或功能回退，才适合归并。

## 待验证来源级风险

1. “未新增原文外事实”的可选说明在入口、工作流和复核表中均出现，可能被弱模型理解为默认尾句。
2. “未形成结论时保留尚未形成结论”的任务卡措辞，需要持续区分用户明确给出的中间状态与材料没有提供结论。
3. 首尾位置核对、待确认提示、正式报送缺口和引用核验提醒重复出现，长任务中可能被复述成制作型说明。
4. “补充以下信息后，文章会更完整”及“影响正式报送/文种完整性”在事实稀疏任务中可能过度展开。
