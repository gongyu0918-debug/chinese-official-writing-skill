# 稀疏材料压力测试与核心工作流复核

## 结论

“材料很少 + 强制 650-800 字 + 反复要求不得补造”是有意制造的约束冲突，只适合作为压力测试，不应单独作为正常写作质量的验收条件。材料只能支持短稿时，未达到字数属于指令冲突下的可解释 WARN；为达到字数而补造主体、流程、措施、成效或责任关系才是严重失败。

本轮没有修改 canonical `SKILL.md` 或任何写作 reference。此前尝试的短审核链路和大范围 Markdown 清晰化候选均已撤回，原有“小段写完先审、小节写完再审、全文合并后做总审”流程保持不变。

## 历史版本复核

固定版本：

| 版本 | commit | 已存在的相关规则 |
| --- | --- | --- |
| 1.4.10 | `d8d7ee0a96a0c0b822446bd70d1d417eef2a729e` | 事实不足不默认中断，先完成正文，再列正文外缺口。 |
| 1.4.12 | `92f6d15cb8802d1e66d8fad6f8d775db6945d9c4` | 延续“先成稿、不补强结论”，但曾推荐“从已给材料看，问题集中于……”。 |
| 1.5.2 | `70efc9ce74fe956497b5044ee14f60e2b94c5e55` | 延续材料不足先成稿和不补造规则；真实写稿仍可出现材料读取旁白。 |
| 1.5.7 | `d3755df7deb2456150c61ccb1944aa3982f7edf1` | 明确事实少于目标字数时宁可短写；删除容易诱发旁白的旧推荐句。 |

历史文本说明，这类冲突并非 1.5.x reference 拆分后新出现。1.4.10 已同时存在“材料不足仍先成稿”和“不补事实”的张力；1.4.12 又加入了容易被弱模型直接写入正文的材料视角句。1.5.7 对冲突优先级的处理更明确，不能据此压力样本倒推核心流程退化。

## 真实写稿对比

同一 Luna 低思考 writer 对 1.4.10、1.5.2、1.5.7 和本轮候选分别执行两项任务：

1. 仅给三项事实，强制 650-800 字，只输出正文，不得补造。
2. 同样三项事实，允许写成简短情况说明，不得凑字。

两名独立 blind verifier 的共同结论：

| 版本 | 强制长稿 | 正常短稿 |
| --- | --- | --- |
| 1.4.10 | 为拉长篇幅出现重复、材料读取说明和自证语句，伴随轻微推断。 | 短而干净。 |
| 1.5.2 | 重复和材料旁白更明显，并补入记录、整改、效果等未给动作。 | 短而干净。 |
| 1.5.7 | 未达到 650 字，但保持已给事实，没有为凑字补造。 | 短而干净。 |
| 本轮候选 | 与 1.5.7 一致，未证明写稿能力另有提升或退化。 | 短而干净。 |

因此，把“强制长稿未达字数”直接判为功能失败属于过严口径。正确裁决是：先保护事实；若交付模式允许，在正文外简短说明材料与篇幅冲突；用户要求只输出正文时不附说明。

## 核心 workflow 消融

曾实验以下候选：段落只做四项极短检查、小节不再完整复核、全文只做一次汇总审核、最多局部修订一次。真实 A/B 没有证明其优于 1.5.7，候选低思考样本还出现：

- 未删除用户点名要求删除的过程说明。
- 把明显自证旁白误判为可保留。
- 一次无关外文残片 `നിലവ`。按当前标准，外文思考或无关残片出现一次即构成严重回退。

两名 blind verifier 均未支持保留该候选。短审核链路已完整撤回，未进入 canonical、镜像或发行包。为防止后续文字精简静默改变流程，新增结构测试，锁定分段、小节、全文三级复核顺序。

## Markdown 清晰化候选

13 个 Markdown 文件的广泛去重和语言精简候选也参加了短样本 A/B。候选中等思考与基线大体相当，低思考却出现上述指令遗漏、误判和外文残片，无法证明稳定提升。该候选已整体撤回，不在失败版本上继续叠补丁。

这不否定“使用自然、完整、精炼中文”的方向，只说明应按单个小块逐次修改和验证，不能把视觉上更简短直接等同于模型理解更稳定。

## 本轮保留改动

- `prose_lint.py` 的交付检测仍为显式启用、只读提示，不进入默认写稿链路，不自动删除或改写正文。
- 明确的约束自证、英文思考残片和交付说明仍为 high；材料范围句降为 medium，交给人工结合来源判断，避免把调查范围、审计边界等正常业务表述误杀。
- 补充 `已有材料`、`已根据用户要求完成修改`、`I will ...` 和 Markdown 形式正文外提示标题的检测覆盖。
- `AGENTS.md` 增加核心工作流变更保护规则；测试新增原分层复核流程守卫。
- 独立 cold review 首轮发现四个 high 误报：正文外允许区的事实边界短句、普通业务句“已根据要求完成调整”、普通英文句“I will attend the meeting”和业务标题“风险提醒机制建设情况”。四项均已复现并收窄；检测器继续命中“已根据用户要求完成修改”和“I will draft the final report”等明确交付旁白。
- 同一 reviewer 复查确认 `resolved=true`、`publish_blocking=false`，未发现 canonical 或三级复核流程变化。

## 验证结果

- 定向 unittest：`129/129` 通过。
- 全量 unittest：`142/142` 通过。
- v1.5.7 确定性消融：baseline `102/103`，current `103/103`；新增差异仅为 P104 可选检测模式。
- Promptfoo smoke：首次受失效 Hermes Python 路径和沙箱可见性影响，20 项均为启动错误；使用 Codex bundled Python 并在沙箱外重跑后 `20/20` 通过，judge consistency `1.0`。
- 真实文章结构回归：skill 关键要素 `61/61`；stub 仍有 9 个占位词风险样本，该指标不代表真实 writer 质量。
- `quick_validate.py`：通过。
- `tools/sync_adapters.py`：通过，镜像与 canonical 对齐。

## 剩余风险

- 检测器是正则和模式提示，不能替代语义判断；材料范围句必须结合原始材料判断。
- 本轮没有保留 prompt/reference 改动，因此没有再跑 3000 字长文矩阵；按测试分级规则，当前改动不触及默认写稿链路，短 A/B 已足以裁决被否决候选。
- 不能从一次外文残片证明其由某一句 Markdown 引起，只能证明该整体候选未达到保留门槛。
- 稀疏材料同时要求长篇时，事实保真与篇幅达标仍存在客观冲突；这不是靠补充禁令即可消除的问题。
