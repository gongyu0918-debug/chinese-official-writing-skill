# 多轮改稿“后一轮撤销前一轮修改”实测

日期：2026-07-15。

## 目的与边界

验证修改模式能否只撤销上一轮中被点名的变化，同时保留本轮明确要求继续生效的内容。测试覆盖选择性恢复、恢复被删段、取消整项及关联内容、删除正文外提示四种路径。未修改产品 Prompt，也未把测试标签写入成稿。

## 方法

- 两名独立 writer 先完成第一轮改稿；第一轮输出锁定后，分别收到相同的第二轮撤销指令。
- 第二轮只允许读取各自第一轮成稿和撤销指令，不读取另一名 writer 的输出或 reviewer 结论。
- 两名独立 reviewer 只看原始任务、第一轮成稿、第二轮指令和第二轮成稿，按选择性恢复、关联项清理、保留项稳定、事实漂移判断。
- 第三名 verifier 复核全部原始证据和两名 reviewer 结论。

## 结果

| 场景 | Writer A | Writer B | 结果 |
|---|---|---|---|
| V1 恢复原接收单位和关联指代，保留新期限 | PASS | PASS | 职责和指代恢复，9月10日、9月12日及信息技术部职责保留 |
| V2 恢复上一轮删除段，保留新标题和章节调序 | PASS | PASS | 经费测算段恢复，新标题和章节顺序保留 |
| V3 取消试行整项和关联日期、职责、周期、材料，保留联系人修改 | PASS | PASS | 试行链条清除，联系人李宁、12345678保留 |
| V4 删除正文外待确认提示，保留正文期限 | PASS | PASS | 文后提示清除，10月12日保留 |

合计 8 PASS、0 WARN、0 FAIL。两份第二轮成稿内容一致；机械扫描未发现旧单位、旧电话、试行日期、周报周期、评估材料或待确认标题残留。两名 reviewer 的 8 项结论均获第三名 verifier 确认。

## 修复判断

没有出现同一语义问题覆盖至少3份输出、2个 prompt、2名 writer 并经 reviewer 多数确认的失败。本轮不修改产品 Prompt。

## 剩余边界

- 第三轮及以上反复撤销、恢复和再次修改。
- 上下文压缩后对早期版本的回溯。
- 第一轮已存在漏改时，第二轮同时纠错和撤销。
- 长文跨章节、附件引用、表格、字段式材料、DOCX 样式和复杂模板。
- 范围含糊或互相冲突的撤销指令。
- 跨模型条件下不同输出路径的稳定性。
