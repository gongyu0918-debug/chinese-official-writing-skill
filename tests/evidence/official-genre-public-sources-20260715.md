# 六类薄弱公文政府官网原文对照实测

日期：2026-07-15。

## 目的与门禁

针对决定、复函、批复、公告、公示、致辞六类此前覆盖偏薄的文种，以政府官网公开原文为事实和文种参照。writer 不读取网页原文，只收到“标题 + 自然语言事实 + 交付要求”；reviewer 再把成稿与公开原文对照。只有同类问题至少出现在3份输出、2个 prompt、2名 writer 中，并经独立 reviewer 共同确认，才进入语义层最小修复。

## 样本来源

- G1 决定：北京市人民政府门户网站转载的表彰决定。
- G2 复函：上海市人民政府主动公开公文库的领事认证费复函。
- G3 批复：上海市人民政府门户网站的国土空间总体规划批复。
- G4 公告：喀什地区行政公署门户网站的延长报名时间公告。
- G5 公示：西藏自治区民族事务委员会的正式推荐对象公示。
- G6 致辞：中华人民共和国外交部发布的论坛开幕式致辞。

来源链接、访问日期、事实指纹和原文结构见 `tests/evidence/official-genre-public-sources-20260715/source-notes.md`。

## 方法

- 两名独立 writer 各完成6份成稿，共12份。
- 两名独立 reviewer 分别检查事实保真、文种功能、必要结构、自然度和成品外防卫旁白。
- 第三名 verifier 逐一打开六个政府官网来源，复核12份成稿及 reviewer 分歧。
- 不要求逐字复刻原文固定话语；只比较题面事实、行文关系、文种功能和正式程度。

## 结果

| 文种 | Writer A | Writer B | 第三方核验 |
|---|---|---|---|
| 决定 | PASS | PASS | 2 PASS |
| 复函 | PASS | PASS | 2 PASS |
| 批复 | PASS | PASS | 2 PASS |
| 公告 | PASS | PASS | 2 PASS |
| 公示 | PASS | PASS | 2 PASS |
| 致辞 | PASS | PASS | 2 PASS |

第三方总判定为 12 PASS、0 WARN、0 FAIL。12份成稿均未出现“未新增事实”“按要求”“待确认”“仅依据材料”等成品外防卫旁白。

两处分歧经第三方复核后不构成缺陷：

- 两份公告均省略“特此公告”。原文使用该句，但标题、变更事项、适用对象、办理规则、联系方式、附件、署名和日期完整，省略程式性结语未造成公告功能缺失。该现象只有2份输出、1个 prompt，且 reviewer 有分歧。
- 一份致辞使用“深入讨论”“具有重要意义”“相互尊重、相互支持”。这些属于场合评价和条件式倡议，未补造会议成果、政策决定或具体承诺；官方原文也有相近语境支撑。该现象只有1份输出、1个 prompt。

## 修复判断

没有任何问题达到共性门槛，本轮不修改产品 Prompt。六类中公告链路相对更值得继续观察，但当前只能记录为测试线索，不能据此把“特此公告”改成硬模板。

## 剩余边界

- 每类只有1个 prompt，不能代表全部子场景。
- 事实输入较完整，未覆盖主体、日期、依据或办理要素缺失。
- 只覆盖一次起草，未覆盖六类文种各自的改稿、只审不改和多轮事实锁定。
- 公告仍应补一个不同主题、明确要求完整正式格式的样本；致辞可补一个事实较稀疏样本，观察礼仪性展开是否升级为政策承诺或会议成果。
