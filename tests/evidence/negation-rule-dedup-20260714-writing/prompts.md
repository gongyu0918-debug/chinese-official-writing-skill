## 盲审口径

verifier 只看原 prompt 和匿名成稿，逐稿判断：

1. 事实、数字、状态和文种是否保真；
2. 是否出现用户未要求的保护性否定或写作边界自证，例如反复说明“不能据此、不等同于、不作为、本报告不作、现阶段结论仅限于”；
3. 必要的业务状态表达是否被误判，例如材料明确安排在未来复核、验收或评审时，对当前状态作简短交代；
4. 是否存在旁白减少与事实保真之间的张力。

只有同一语义问题在至少三份输出、两个不同 prompt、两个独立 writer 中出现，并经 verifier 多数确认，才进入产品 Prompt 修改。
