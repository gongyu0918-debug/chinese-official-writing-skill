# 历史版本漂移消融预注册

## 目的

检查 `chinese-official-writing` 从 1.2.x 到 1.5.14 的版本迭代，是否在提高事实边界的同时造成篇幅不足、显式事实遗漏、边界句重复、语言自然度下降或过度约束。无 Skill 是固定基线，历史版本之间只用于定位漂移起点。

## 第一阶段：GPT-5.5 补充定位拐点

模型固定为 `gpt-5.5`、medium。每个条件使用独立上下文，只完成 `tests/evidence/no-skill-risk-ablation-20260715/prompts.md` 中 T02、T04，不读取判分标准、其他版本输出或历史结论。

GPT-5.5 只作历史版本补充定位，不作为当前质量裁决主模型。它用于缩小需要由新一代模型复核的版本范围，也用于发现“版本变化后才出现”的断点线索。

## 第二阶段：LUNA / TERRA 主测

主测固定使用 `gpt-5.6-luna`、`gpt-5.6-terra`。N 与当前 1.5.14 均使用相同宿主外壳、仅输入原始任务的 T01—T04 新直写稿；旧有带测试控制语的稿件只作诊断，不进入严格配对。历史 harness 组合、1.5.2、1.5.3 和当前 harness 组合另行补测。GPT-5.4 只在 LUNA / TERRA 对旧模型依赖出现分歧时作为补充，不与主测同权。

1.2.x 的历史条件不能只测裸 Skill。用户确认当时主要搭配 `official-writing-agent-community` harness 使用，因此主测至少拆成无 Skill、harness-only、harness + 历史 Skill、当前原生 Skill、harness + 当前 Skill 五类条件。具体预注册见 `new-model-harness-plan.md`。

不沿用历史 5.4/5.5 分数。历史记录只用于选择版本和风险面，本轮必须重新生成同题输出。

## 固定观察项

1. **事实覆盖**：T02 必须保留 `新增1140个账号` 等所有显式数字；T04 必须保留唯一明确议定事项和全部未决事项。
2. **材料外扩写**：原因、职责、后续动作、会议意见、会议要求和状态不得由模型补成已定事实。
3. **篇幅与结构**：按统一非空白字符和中文字符两种口径记录；同时由 verifier 判断后半篇是否完整。
4. **语言自然度**：记录 `不是……而是……`、无锚定否定、`现有材料`、`现阶段`、`尚未形成`、`未形成结论`、`上述` 等显式句式；机械计数不代替语境判断。
5. **直接可用性**：盲审以 PASS/WARN/FAIL 判定，事实、文种、格式和输出模式回退优先于语言偏好。
6. **整体写稿观感**：比较信息组织、段落推进、首中尾平衡、结论力度、阅读顺畅度和直接采用成本；篇幅只占其中一项。
7. **句法完整性**：逐句检查动作主体、谓语宾语和指代是否能由原材料或上下文还原，记录漏主体、漏宾语和对象不明。
8. **机械顺序句**：检查“先……再……”“先行……后……”等顺序表达。真实程序关系保留；同稿成簇重复、没有办理顺序依据或仅为组织句式时记为语言风险。

## 集中修复门槛

- 同一风险至少跨 3 个任务或跨 2 个模型稳定出现，才进入产品修复。
- 先定位版本拐点和首次引入的规则类型，再集中修改重复规则或优先级；不按单句追加新禁令。
- 修复后必须同时对无 Skill、拐点前版本和 1.5.14 修复前版本做同题盲审。
- 事实保真改善若以篇幅、数字保留、文种、格式或自然度回退为代价，不判为通过。
