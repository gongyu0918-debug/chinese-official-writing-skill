# 同模型无 Skill 质量门槛与最小修复预注册

## 目标与版本边界

本轮继续使用 1.5.14 版本号，只修复已被真实写稿复现的共性质量漂移。上一发行版用于检查新增回退；同模型无 Skill 用于判断 Skill 是否产生净质量增益。两类基线分别报告，互不替代。

harness 只作历史诊断，不参与产品胜负、修复触发或发布裁决。Skill 本体不依赖独立 reviewer、额外调用轮次或宿主 agent 调度能力。

## 同模型配对原则

- writer 固定使用 `gpt-5.6-luna`、`gpt-5.6-terra`、`gpt-5.6-sol`。
- 同一任务的无 Skill 与带 Skill writer 使用同一模型、同一 thinking 档、同一任务正文和独立上下文。
- 两组均不读取对方输出、盲审标准、测试结论或公开映射。
- 带 Skill 组只按当前渐进路由读取实际命中的入口和 reference。
- 严格无 Skill 组一题一个新线程，用户消息只包含原始任务全文，不增加“无 Skill、基线、测试、不得读取、不要看其他输出、只凭模型能力”等控制语，也不增加分隔标记或测试说明。
- 创建严格无 Skill 线程时，公文 Skill 不得出现在该线程的可发现技能目录或注册目录中；创建后立即恢复本机安装状态。线程若自报使用 Skill、读取 Skill 文件，或无法证明创建时 Skill 不可发现，该样本作废。
- App 自动生成的 delegation 传输外壳如无法取消，单独记录为平台传输层；外壳内 `<input>` 必须与原始用户任务逐字一致。
- 随机 seed 无法固定时，明确记录为独立上下文配对，不把单次随机差异写成严格因果。

## 领先判定

盲审只看原任务和匿名成稿，先检查事实、文种、格式、输出模式和明确指令，再比较完整性、组织、语言自然度、主体宾语、篇幅均衡和直接采用成本。

只有带 Skill 稿在硬边界不低于无 Skill，并在整体写作质量上被明确判断为更好，才记 1 次严格胜出。平局、各有优劣、两稿都失败或证据不足均不计领先。事实、文种、格式或输出模式出现 Skill 独有回退时，该配对记为硬回退。

开发用 T01—T04 只用于定位和逐项筛选，不计入最终 80%—90% 领先率。最终门槛使用修复前固定的 12 个留出任务，覆盖：

1. 1800—2200 字多材料阶段报告；
2. 600—800 字稀疏异常报告；
3. 900—1100 字部分已决、部分未决会议纪要；
4. 700—900 字请示或申请；
5. 800—1000 字通知；
6. 600—800 字函或复函；
7. 1500—1800 字方案；
8. 300—500 字字段式材料改写；
9. 300 字以内保要素压缩；
10. 只审不改；
11. 以最新版为主线的第二轮修改；
12. 1200—1500 字总结或工作要点。

每个模型形成 12 个同模型配对，共 36 个配对。发布最低线为每个模型至少 10/12 严格胜出，且总计至少 30/36；目标线为每个模型至少 11/12、总计至少 33/36。全部配对的 Skill 独有事实、文种、格式和输出模式硬回退必须为 0。未达到最低线时，不更新公开对比节选，不发布，不用局部样稿代替总体结果。

## 逐项最小修复顺序

每次只处理一个共因，形成独立 commit 和同模型 A/B。出现事实、文种、格式或输出模式回退时立即撤回该候选，再换方法。

1. **候选 A：解除过度压缩优先级。** 先只调整 1.5.3 引入并沿用的“事实少于字数目标时宁可短写”硬边界，把目标改为“事实不外扩，同时完整覆盖显式事实并完成文种功能”。不改路由、reference 加载条件、复核轮次或输出模式。
2. **候选 B：减少同向规则叠加。** 若候选 A 仍不能恢复篇幅和整体观感，再合并入口、workflow、playbook 中语义重复的禁补清单；保留总事实边界和必要文种特例。
3. **候选 C：补显式事实覆盖自检。** 若仍出现数字、日期、主体、对象、未决状态遗漏，在同一上下文的成稿复核中增加正向覆盖核对，不增加外部 reviewer 或新调用阶段。
4. **候选 D：修复纪要归属与语义升级。** 只有跨模型仍反复出现会议意见、部门职责、程序安排或主体宾语无锚点时，才收紧“谁提出、会议决定了什么、动作指向什么”的语义检查；不禁用“先……再……”等正常顺序词。

旧版本中的单条规则如在同模型 A/B 中更优，可以做规则级回滚；不整版回退。1.5.14 已验证为正向的渐进路由、文种仲裁、格式边界、用户模板优先和输出模式继续保留，并在每个候选后复测。

## 每个候选的验证顺序

1. 先跑与改动直接相关的 LUNA、TERRA T01—T04 同模型短矩阵。
2. 盲审确认无硬回退且方向改善后，再跑 SOL 同题配对。
3. 同步运行上一发行版确定性消融、全量单测、Promptfoo smoke、真实改稿和旧能力回归。
4. 只有最终候选进入 12 个留出任务、3 个模型、36 个配对的发布级矩阵。
5. 所有 writer 原始稿、匿名包和 verifier 原始结果先提交；映射与汇总后提交，保留揭盲顺序。

## GitHub 对比展示

严格裸基线与自然触发当前 Skill 的同模型配对，经盲审确认整稿全面领先且没有事实、文种、格式或输出模式回退时，可以先替换 GitHub 对比展示。展示同时满足：

- 原始用户 prompt 逐字放入代码块；
- 实际 writer 模型放入代码块，并标明 thinking、独立上下文和非同一随机 seed；
- 无 Skill 与带 Skill 成稿均来自上述同模型配对，不把受控无 Skill 旧稿混入；
- 节选周边结论与整稿盲审一致，并链接完整原稿、thread、匿名包和 verifier 结果；
- 没有全面领先样本时继续保留风险记录，不为展示目的另挑或改写基线稿。
