# GPT-5.5 历史版本漂移阶段结论

> 2026-07-15 证据更正：GPT-5.5 的“无 Skill”线程带有“不得使用 Skill、不得读取历史输出或判分标准”等控制语，不是严格裸基线。1.5.2→1.5.3 的同模型历史版本断点不依赖该组，仍作为版本漂移线索保留；涉及无 Skill 的胜负、失败率和净收益判断全部降级为受控诊断，等待只含原始任务的裸基线重跑。

## 已完成范围

- 同一模型：`gpt-5.5`、medium。
- 同一任务：T02 多材料情况报告、T04 未决事项会议纪要。
- 九个独立上下文：无 Skill、1.2.5、1.2.18、1.2.29、1.4.8、1.5.2、1.5.3、1.5.5、1.5.14。
- 盲审：`gpt-5.6-terra`、high，只读取匿名稿、原任务和预注册观察项。

## 可以确认的漂移

1. **1.5.2→1.5.3 是篇幅断点。** T02 从 1119 CJK / 1386 NonWS 降至 901 / 1138，T04 从 895 / 1031 降至 700 / 824；1.5.5 仍处低位。该变化同时出现在两个任务，不是单篇偶然短写。
2. **1.5.14 出现部分恢复。** T02 回升至 1268 / 1537，T04 回升至 900 / 1045；但恢复篇幅没有解决纪要的材料外会议意见问题。
3. **过度压缩没有换来完整事实保真。** 1.5.3、1.5.5 的数字和未决状态齐全，但仍把未记录意见或要求写入纪要，因此“更短”等于“更稳”不成立。
4. **固定口癖不是本轮版本漂移主因。** 18 稿均未出现“不是…而是”；边界词也无单调上升趋势。语言僵化主要体现为“会议听取—会议认为—会议研究—会议明确”的结构链和推进句式。
5. **包体大小不能单独解释结果。** 1.2.18 是早期高密度节点，随后 1.2.19 曾大幅减负；本轮 1.2.18 未出现 1.5.3、1.5.5 级别的共同压缩。更值得核对的是规则类型、重复位置和优先级。

## 与源码 diff 的对应线索

1.5.2→1.5.3 在 canonical `SKILL.md`、`workflow.md`、`review-checklist.md`、`proofreading-checklist.md` 和 `genre-playbooks.md` 同时增加细粒度事实边界。入口硬边界新增“事实少于字数目标时宁可短写”，并多次列举不得补入的主体、流程、产物、范围、责任和后续动作；`workflow.md` 又在事实充分性段复述同向规则。

这能解释模型为何在篇幅与事实安全冲突时更偏向短写，是待复核的机制线索。它还不能解释 T04 为何仍补造会议意见，也不能凭单模型证明这些新增规则必然导致漂移。

## 暂不修改产品的理由

预注册门槛要求同一风险跨 3 个任务或跨 2 个模型稳定出现。当前只完成一个模型、两个任务；盲审也明确要求不得据此触发产品层修复。下一步以 LUNA、TERRA 为主，拆分无 Skill、harness-only、harness + 历史 Skill、1.5.2、1.5.3、当前原生 Skill 和 harness + 当前 Skill。只有新一代模型结果仍指向同一共因，才集中调整重复规则或优先级。

本轮也不能更新 README 为“当前 Skill 稳定优于无 Skill”：无 Skill 在两题均失败，当前 1.5.14 只在 T02 获得 WARN，T04 同为 FAIL。公开节选必须等集中修复和复测通过后再选择。
