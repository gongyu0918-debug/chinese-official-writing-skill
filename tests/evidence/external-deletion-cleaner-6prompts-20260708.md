# 外部删减型清稿链路：6 prompt 弱稿修复测试

日期：2026-07-08

目的：验证不继续向 `SKILL.md` 堆入口规则的替代方向，即把弱模型失败稿交给外部删减型清稿链路，观察是否能把典型 FAIL/WARN 拉回可用。本轮不修改 skill，不把该链路写入默认 workflow。

## 方法

- 输入：上一轮弱模型失败稿，覆盖 6 个真实 prompt。
- cleaner：`gpt-5.5`，低思考，加载 `chinese-official-writing` skill。
- cleaner 约束：只根据“原始 prompt + 待清稿”删减，删除未授权事实、管理链条、Markdown、占位和日期变形；还原用户给定动作和时间；不补新内容，不输出问题清单。
- verifier：`gpt-5.5`，低思考，只看“原始 prompt + 清稿输出”，不重写稿件。

## Verifier 结果

| 场景 | 结果 | 判断 |
| --- | --- | --- |
| P1 电脑采购说明 | PASS | 5090、35B 小模型、版慎通、harness、本地脱敏、一次性投入、OCR、文生图等要点保留，删除占位和补造信息。 |
| P2 微博情况说明 | PASS | 日均发送量、阅读量、粉丝数、增长率、阅读高峰、本地新闻弱和下一步投放均保留，无额外机制。 |
| P3 Claude Code 排查通知 | WARN | 保留排查、删除、无也汇报、`7月10日15:00`、钉钉、技术应用部、单位和日期；但正文外提示放在落款后，可能被误认为正式内容一部分。 |
| P4 气象文章改通知 | WARN | 通知可用，落款正确；但如果原气象文章已经给出具体风险，清稿可能删掉来源材料关键事实。 |
| P5 世界杯期间通知 | PASS | 保留禁止赌球、考勤、祝观赛愉快、发送方和发送对象，删除处罚、检查、反馈流程等补造链条。 |
| P6 OpenClaw 排查报告 | PASS | 保留报告对象、发送方和“未发现安装”结论，删除排查范围、技术细节和后续管理机制。 |

总体：`PASS with WARN`。

## 结论

外部删减型清稿链路比“入口事实授权短检”更有效，能够把弱模型失败稿中的未授权管理链条和 Markdown 残留基本删干净，特别是 P6 从 FAIL 拉回 PASS。

但它还不能直接作为发布结论：

1. 正文外提示需要明确隔离，不能紧跟落款日期而像正式正文。
2. 来源文章改写场景要防止过度删短，不能把原文中已经给出的关键气象风险、版本范围、名单或日期删掉。

下一轮建议只压测这两个风险，不扩大成默认写作阶段，也不把 cleaner 写进 `SKILL.md`。
