# 无 Skill 风险消融阶段报告

> 2026-07-15 证据更正：本轮所谓“无 Skill”writer 收到过“不得读取 Skill、不得看其他输出、只凭模型能力”等测试控制语，不属于只含原始用户任务的严格裸基线。下列成稿、机械测量和盲审继续保留为受控诊断证据；所有“相对无 Skill 胜出/落后”的产品结论均须由严格裸基线重跑后重新判定。

## 结论

本轮 4 个任务在写作前固定，LUNA、TERRA 分别完成无 Skill 和 1.5.14 带 Skill 写作，共形成 16 份正常路由稿；另用 LUNA 生成 4 份强制完整路由稿。writer 为独立上下文，不读取判分项或对方输出。

当前 1.5.14 显示出明确的能力交换，而不是对无 Skill 的稳定全面优势：

- 带 Skill 稿通常更少把材料外原因、职责、处置和会议决定写进正文。
- 带 Skill 稿在两个模型的四个任务中逐题短于无 Skill 稿；LUNA 强制完整路由又逐题短于正常路由，说明问题不只是轻量卡误配。
- TERRA 带 Skill 的 T02 遗漏了用户明确要求保留的“新增1140个账号”，并低于篇幅下限；同题无 Skill 稿保留该数字并达到篇幅要求。
- T04 中，两模型无 Skill 稿都把材料未给的办公室、财务处职责写成会议安排；带 Skill 稿没有扩写这组职责，但仍有保护性说明或建议状态提升问题。
- 20 份稿均未出现 `不是……而是……`；口癖、无锚定否定和 AI 固定句式没有形成带 Skill 优势。带 Skill 稿的“现有材料”“现阶段”“尚未形成”“上述”等边界短语反而存在重复。

因此，当前 README 不应把单个节选写成整体质量胜出。可确认的局部优势是事实边界；已确认的风险是篇幅偏短、显式事实遗漏、边界句重复以及完整路由加剧压缩。

## 路由消融

LUNA 在相同四题上强制读取完整相关路由，非空白字符从正常带 Skill 的 `1308/1337/577/647` 降为 `988/979/416/512`。完整路由没有改善篇幅，反而进一步压缩。

这一结果排除了“只要强制走最大路由即可恢复质量”的简单解释。更可能的原因是：入口和 references 中多次重复事实边界、缺项边界和未决状态规则，而安全展开、篇幅预算和后段充分性缺少同等强度的正向动作。加载更多同方向规则会提高保守约束权重。

## 当前风险

1. **过度压缩**：两个模型都复现；强制完整路由更明显。
2. **事实保真与内容充分性的张力**：带 Skill 减少材料外扩写，但 TERRA T02 出现显式数字遗漏。
3. **边界句口癖**：带 Skill 未稳定减少固定句式，TERRA 组的相关短语更多。
4. **评测目标漂移**：长期发布门禁主要比较上一版本，能检出新增硬回退，却不能证明相对无 Skill 的净收益。
5. **公开证据风险**：局部节选可能掩盖整稿相反表现；README 更新必须等跨版本和跨模型结论稳定后再选稿。
