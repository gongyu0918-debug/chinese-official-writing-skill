# 无 Skill 风险消融预注册

> 事后审计说明：本计划执行时给无 Skill writer 增加了“不得读取 Skill、不得看其他输出”等控制语，导致该组不满足严格裸基线定义。原计划与结果留作历史记录；新的裸基线规则见 `../historical-drift-20260715/no-skill-quality-gate-plan.md`。

## 目的

本轮不是为 README 挑选单个优势案例，而是重新观察当前 1.5.14 与无 Skill 写作的真实差异，并检查长期 Prompt 迭代是否带来过度约束、解释腔、事实机械复述、结构失衡或自然度下降。

## 固定设计

- 写作模型：TERRA、LUNA。
- 每个模型分别运行无 Skill 和带 Skill 两个独立上下文。
- 两组接收完全相同的 4 个真实用户式任务；无 Skill 组不读取本仓库 Skill，带 Skill 组读取 1.5.14 canonical `SKILL.md` 和任务命中的 references。
- writer 只收到任务，不读取判分项、另一组稿件或既有评测结论。
- 每组必须交回全部 4 稿，不因质量好坏删样本或重跑；工具故障、空返回和截断单独记录。
- writer 完成后按模型内随机映射为 A/B，交给未参与写稿的独立 verifier。verifier 只看原任务和匿名稿件。
- README 只有在完整 8 组配对中出现可复核差异时才引用样稿；单个幸运样本不用于概括整体优势。

## 观察项

### 硬边界

1. 已给主体、日期、数量、比例、金额、时限、未决状态和标题顺序是否保留。
2. 是否新增原因、责任单位、处置结果、采购决定、推广决定、会议结论或其他未给事实。
3. 文种、行文关系、标题、正文和用户明确输出模式是否正确。
4. 是否满足篇幅要求；机械字数只作辅助，明显截断或大幅偏离单列。

### 写作质量

1. 长稿开头、主体、后段措施或结论是否都有实质内容，后半篇是否因前文铺陈过多而草草收尾。
2. 关键事实是否进入对应段落，是否被概括词替代或在重复改写中遗漏。
3. 否定判断是否能对应材料中的事实、状态或必要边界。
4. 是否自然生成无事实锚点的否定句、`不是……而是……` 式虚假对照、连续保护性说明、规则自证或写作过程旁白。
5. 是否存在同一事实反复解释、章节过密、待确认项过度展开、语言过度谨慎或正式度损失。

## 配对判断

- 每个模型、每个任务单独给出 A/B 的 PASS、WARN 或 FAIL，并列明事实覆盖、未支持内容、结构、语言和直接可用性。
- 带 Skill 稿只有在没有新增硬边界回退时，才能记为质量改善。
- 带 Skill 独有的事实遗漏、事实扩写、文种/格式/输出模式错误均记为回退。
- 带 Skill 独有的解释腔、机械复述、过度分节、篇幅失衡或不自然表达记为过度 Prompt 工程风险；同类风险在 3 个以上配对中出现，才建议修改 Skill。
- 无 Skill 独有风险也按完整样本计数，不以单句或单稿直接推广为模型共性。

## README 取样门槛

对外节选应同时满足：

1. 引文来自本轮原始输出，逐字保留；
2. 同一任务至少在 TERRA、LUNA 两个模型中呈现同方向差异，或明确标为单模型示例；
3. verifier 已核对引文周边上下文，节选不掩盖整稿中的相反表现；
4. 同时披露完整配对结论和带 Skill 风险，不把真实消融写成单向宣传。
