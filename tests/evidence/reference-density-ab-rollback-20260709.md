# 2026-07-09 reference 信息密度拆段 A/B 回滚记录

## 目的

用户提出压缩 prompt 时要关注原始文档和压缩后文档的信息熵：原文包含专业术语、非自然语言片段、解释不清片段和潜在重复片段，应一步步最小压缩，以提升弱模型性能，同时保证强模型写稿质量不回退；也可以优化渐进式路由，让每次写稿按最小需要加载 reference。

本轮不继续压缩 `SKILL.md` 入口事实边界。此前已有三轮证据显示，入口瘦身、长句拆行和短原则转 reference 均未通过真实 A/B，容易伤害弱模型事实关系。

## 只读定位

本地长行和重复锚点排查显示，当前最明显的高密度片段是 `chinese-official-writing/references/workflow.md` 的“事实充分性软处理”段落。该段把缺项处理、上行文结构缺口、正文缺项表达、强判断边界、事实外扩、playbook 边界和后续轮次软提示揉在同一长段里。

独立 explorer 只读审查也建议：如继续做注意力优化，优先尝试把 `workflow.md` 第 95 行附近的事实充分性长段拆成小标题块；不能压缩 `SKILL.md` 的具体硬边界、字段式材料细节、联网边界、Markdown 残留规则和渐进式加载表。

## 候选修改

临时候选只改 `workflow.md` 及同步镜像，未改 `SKILL.md` 入口、脚本、lint、版本号或发布元数据。

候选把单个“事实充分性软处理”长段拆成 7 个短块：

- 缺项处理
- 上行文结构缺口
- 正文缺项表达
- 强判断边界
- 事实外扩边界
- playbook 边界
- 后续轮次

语义目标是只降低单段密度，不删除历史回归锚点。

## 真实写稿 A/B

### Prompt 覆盖

同一批 prompt 同时跑当前基线和候选版，弱模型使用 `gpt-5.3-codex-spark low`，强模型使用 `gpt-5.5 low`。

1. 版慎通模型成本优化情况说明：检查 6 万元/月、贵价模型、无 token-plan、低思考档位、防滥用、模型路由、按付费分级是否落实，且不写成已决定执行。
2. 内部账号清查通知压缩到 260 字以内：检查联系人、电话、OA 反馈渠道、落款和日期是否保留，不使用 Markdown。
3. 当前微博情况说明：检查 200 条/日、100 万阅读、500 万粉丝、增长 5%、21-23 点、社会/娱乐热点、本地新闻弱和“拟/计划/建议”口径是否保留，不补责任部门和完成时限。

### 基线结果

- weak baseline：独立 verifier 判定 PASS。三篇要点基本齐全，A 保持建议口径，B 满足 260 字压缩和关键要素保留，C 仅有轻微包装语。
- strong baseline：独立 verifier 判定 PASS。A、B、C 均能保留关键事实和建议口径，无 Markdown，无明显事实外扩。

### 候选结果

- weak candidate：独立 verifier 判定 FAIL。
  - 三篇标题均出现 `**...**` Markdown 加粗，且自查误称无 Markdown。
  - A 新增“跨模块复核明确分工”“近三个月调用结构”“预算监测报表”“按月复盘”等未给管理动作。
  - B 增加“请各部门认真落实”，把压缩通知推向更硬执行要求。
- strong candidate：独立 verifier 判定 WARN。
  - A、B 基本通过。
  - C 增补“本地民生、公共服务、城市动态”等具体选题方向，属于轻度内容外扩。

## 结论

候选不保留，已撤回全部产品文件改动。

本轮说明：即便只拆 `workflow.md` 按需 reference 的超长段落、且定向单测通过，也可能让弱模型在真实写作中更容易出现 Markdown 残留、事实外扩和建议口径执行化。后续不能把“更短、更分块、更像清单”直接等同于更好。

## 后续建议

- 暂停压缩 `SKILL.md` 入口事实边界，也暂停无语义拆分 `workflow.md` 事实充分性长段。
- 若继续做信息熵优化，先只做只读标注，不写产品文件；每次只提出一个候选，并固定跑弱/强模型真实 A/B。
- 更值得尝试的是加载路径测试：让 writer 明确在同题下分别只读入口、入口 + workflow、入口 + workflow + 命中文种 playbook，观察“多读 reference”是否改善或损害事实边界。
- 若弱模型首稿仍不稳定，优先继续验证二次修改链路和外部只读 final-draft inspection，而不是继续堆 prompt 或拆 prompt。
