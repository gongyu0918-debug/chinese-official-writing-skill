# 1.5.12 发布候选真实写稿 Prompt

以下 4 个 prompt 由两名当前候选 writer 和一名固定 `v1.5.11` baseline writer 独立执行。writer 不得修改仓库。子上下文未返回可核验的精确模型 ID，因此本组只记发布级真实写稿 sanity，不宣称指定模型矩阵或统计显著性；未使用 MiniMax。

## P1 稀疏事实报告

请根据以下事实写一份300字以内的系统巡检情况报告，只输出正文：7月巡检服务器16台，发现3台日志备份延迟，已于7月12日恢复；监控模块已上线，检索模块仍在本地测试。不要写“未发现其他风险”，不要增加原因、责任部门和整改机制，不要附写作说明。

## P2 只审不改

只审不改，逐项指出下段中的制作说明、无依据否定、虚假对比和机械重复；保护真实状态和必要否定，不给改稿：“本稿为修改版，供领导查看。审校模块已上线，检索模块仍在本地测试。不是一般优化，而是全面跃升。根据制度要求，未通过复核不得发布。未发现任何风险。项目持续推进、持续完善、持续提升。”

## P3 字段式申请

请把以下字段式采购申请改得正式，只输出字段式正文，不增加标题、主送、落款或说明：申请事项：采购扫描仪；用途：档案数字化；数量：2台；预算：单价3600元，总额7200元。不得补品牌、供应商、采购方式或验收安排。

## P4 最新版、旧版与参考样文

最新版事实为“6月完成数据目录核验48项，发现名称不一致4项，已于6月30日修正”。旧版写有“5月核验40项、联系人王明、建立长效机制”，参考样文写有“总体运行平稳、未发生重大风险”。请按参考样文的结构写一份200字内情况说明；不要带入旧版和样文事实，不要联网，只输出正文。
