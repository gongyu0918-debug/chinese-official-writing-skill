## 三次以上共性问题

1. 弱模型事实外扩仍存在：P1、P4、P6 均出现未给性能判断、地域口径或后续管理链条。
2. 弱模型正式格式完整性不稳：P3 落款单位推断，P4/P6 缺落款日期。
3. 弱模型关键要求漏入正文仍会发生：P3 漏受影响版本，P5 漏“禁止赌球”，其中 P5 构成交付失败。
4. 强模型整体更稳，但仍会轻微补写流程、反馈字段或主送对象：P1、P3、P4 均有轻微外扩。

## 当前可用程度判断

当前 skill 在强模型低思考下可作为“辅助起草、需人工核对事实边界”的可用口径；不适合宣称弱模型低思考下已稳定可发布。若发布说明面向真实用户，建议避免表述为“自动可靠生成正式公文”，更准确的口径是：能提高初稿效率，但涉及版本号、主送/落款、排查范围、事实来源和处置要求时仍需复核。

本轮不做 prompt 修复。原因是上一轮已经多次验证，继续堆叠同类场景限制不能稳定解决弱模型首稿问题，且可能造成 prompt 膨胀。下一步若继续修复，应优先考虑：

- 只针对“关键要求漏入正文”做极小范围的写前要点复核提示，而不是新增场景禁令。
- 或继续验证“交付修订模式”作为二次修改路径，让用户指出问题后能稳定删补造事实、补回漏项、清理格式痕迹。
- 任一候选修复都必须重新跑上一基线消融、真实 writer/verifier 复测；若真实复测仍失败，应回退。
