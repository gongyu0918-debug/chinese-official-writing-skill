# 1.5.18 Hook 安全删除契约预注册（2026-07-20）

## 已验证阻断

R18-01 的四处同族风险已被检测器定位。两次协议有效的局部修订均通过机械检查，但语义层安全选择 D0：当纯材料转读或写法自证句只能 `REWRITE` 时，修订 Agent 容易补出重复事实或“总体运行正常”等材料外评价。检测、非空回退和有限状态均正常。

## 唯一变量

只扩展删除动作契约：同时满足以下条件的完整独立句允许 Agent 选择 `DELETE`：

1. 不含数字、日期、数量、主体等硬锚；
2. 整句均为材料转读，或整句由写作边界自证构成；
3. 不是材料明确载明的原句。

含事实前缀、数字锚或未决状态的混合句继续只能 `KEEP/REWRITE`。脚本不自动删除，Agent 仍须逐项决定，D1 仍须通过机械不变量和独立语义裁决；失败回退非空 D0。

## 验证

- 正例：R18-01 的纯材料转读句和纯自证句开放 `DELETE`；
- 反例：含故障数量、处理事实、未决状态、申报/投标/会议材料等业务复合词仍禁止删除或不命中；
- 全量 unittest、Promptfoo smoke、固定 1.5.17 消融、镜像一致性、quick_validate、`git diff --check`；
- 同一 R18-01 D0 只再运行一次完整事务，并由独立 subagent 修订、裁决和匿名盲审。

若出现误删、事实丢失、材料外新增、篇幅硬回退、空稿、循环或 D1 不优于 D0，则该契约不进入 1.5.18。
