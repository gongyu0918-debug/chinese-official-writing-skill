# 当前 1.5.14 与严格无 Skill 同模型盲审汇总

## 结论

本轮使用真正的直接 Prompt 对照：同一模型、同一 medium thinking、同一原始任务、独立 projectless 上下文；两组 Codex/AGENTS 宿主文本哈希一致。严格无 Skill 组创建时本公文 Skill 不可发现；当前组自然触发并完整读取哈希固定的 1.5.14 `SKILL.md`。Luna、Terra 各 4 题，共 8 个同模型配对；每个模型包由 2 名独立 high-effort judge 盲审。

没有一题同时获得两名 judge 的“全面领先”且保持零独有硬回退，因此本轮不替换 GitHub 对比展示。个别 judge 的“全面领先”与同题另一 judge 的事实外扩、篇幅回退或相反胜负结论冲突；单独挑出有利判词会高估 current。

按不考虑硬门槛的整稿偏好计，4 题两名 judge 均偏好 Skill，1 题均偏好无 Skill，3 题分歧，即 Skill 共识偏好为 4/8。按预注册严格口径，平局、分歧、两稿都违反明确要求或 Skill 存在独有事实/格式/输出风险均不计领先；本轮没有可用于 80%—90% 声明或公开案例的严格胜出。T01—T04 是开发定位集，不进入最终 36 对发布门槛。

## 其他风险

- current Luna T02 的异文残片是一次硬输出污染，本样本判退；单次异常不单独转化为产品关键词规则。
- current 在纪要中仍可能把材料中的提出事项扩成“会议指出、会议要求”或扩大部门职责；无 Skill 的同类外扩通常更重，但 Skill 不能因相对较轻而免除事实边界检查。
- “先……再……”没有形成 Skill 独有的稳定劣势；本轮不增加固定禁词。
- 主体、宾语问题仍以局部别扭和职责扩写为主，尚未形成三次以上同形缺失；本轮不做单例规则。
- README、版本号和 Skill 本体在本轮揭盲前后均未修改。
