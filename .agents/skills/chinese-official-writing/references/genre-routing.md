# Genre Routing

Use this reference before drafting when the requested document type is vague, mixed, or likely to be confused with another official-document genre.

## Source Priority

Use this order when deciding the genre:

1. User-provided template, latest draft, or explicit document title.
2. Official genre function from public rules such as the Party and government official-document handling rules.
3. Local unit practice visible in the user's source file.
4. Community templates or examples, only as style hints.

Do not let a community template override the document's function.

## Hard Boundaries

- 请示: used to request instructions or approval from a higher-level authority. Keep one main matter. End with a clear request for approval or instruction.
- 报告: used to report work, reflect situations, or reply to an inquiry. Do not include approval requests or "妥否，请批示" endings.
- 函: used between organizations without a superior-subordinate relationship for consultation, inquiry, reply, approval request within the proper relationship, or business coordination.
- 批复: used by a higher-level authority to reply to a request for instruction. State whether the requested matter is agreed, partially agreed, or not agreed, then list implementation requirements.
- 通知: used to issue, convey, arrange, require, or inform. It should make the recipient, matter, time limit, material, and handling requirement clear.
- 通报: used to commend, criticize, convey important situations, or communicate lessons and requirements.
- 公告/公示: used for public-facing matters. Distinguish broad public notice from limited public display and feedback collection.
- 纪要: records meeting decisions, agreed matters, responsibilities, and follow-up requirements. Do not write it as a news report or speech.

## Routing Hints

- If the user wants approval, funding, staffing, project initiation, or permission, prefer 请示 unless the document is clearly a report attachment.
- If the user wants to summarize progress, explain current status, or report results upward without requesting approval, prefer 报告 or 说明.
- If the text asks another unit at the same level to coordinate, provide materials, reply, or confirm, prefer 函.
- If the text answers a submitted request, prefer 批复 or 复函 depending on relationship.
- If the text assigns work to multiple units with deadlines and materials, prefer 通知 or 实施方案.
- If the text gives a work method, steps, safeguards, and schedule, prefer 方案 or 实施方案.
- If the text argues why a project should be built, funded, procured, or leased, prefer 可研报告、建设方案、采购方案 or 请示附件 according to the user's draft.

## Mixed-Genre Repair

When a draft mixes genres, keep the function that matches the reader's decision need:

- 报告中出现请批事项: move the request into a 请示, or rewrite the sentence as a next-step arrangement.
- 请示中铺陈过多成绩: keep only the facts that support the approval request.
- 通知中写成宣传稿: keep recipients, requirements, deadlines, materials, contacts, and responsibility.
- 方案中只有概念: add target, tasks, implementation path, cost/resource arrangement, responsibilities, and acceptance.
- 技术材料中只有术语: add business demand, resource calculation, cost comparison, SLA, security, delivery, and acceptance.
