# 中文论文开题报告专项链

仅在最终交付物为中文论文开题报告时读取。用户模板、学校要求、题目和最新版底稿优先；本文件不规定固定栏目、章数或篇幅。

## 事实与状态

- 正文只使用用户材料和已核验来源。题目、研究对象、范围、问题、方法、数据来源、进度、条件和预期成果均不得补造。
- 严格区分已有基础、拟开展工作和预期结果；计划不得写成已经实施，预期不得写成已经证实。
- 默认不联网。用户明确要求检索时，只有能够核验并保留引用对应关系的内容才能进入正文。
- 材料不足时先完成能够成立的短稿，缺少的信息放在正文后的构造建议中。

## 由大至小写作

1. 锁定最终要求、研究题目、研究对象与范围、中心问题和可用材料。
2. 为每项内容建立内部对应：需要回答的问题、可用依据、表述状态和证据缺口。
3. 按“中心问题 -> 研究目标 -> 子问题 -> 研究内容 -> 方法与材料来源 -> 进度与已有条件 -> 预期产出”分解任务；该顺序只用于组织思路，正文栏目服从用户模板。
4. 先确定各部分承担的任务，再分解到段落。每段按“本段任务 -> 已有依据 -> 拟采取的研究安排 -> 能够成立的阶段性表述”展开。
5. 方法、进度和可行性只写用户已经给出的方案或条件；尚未确定的内容留在构造建议中。
6. 文献现状只归纳已提供或已核验来源，不从题名、摘要片段或书目信息推测具体发现。

## 正文后构造建议

用户未限制只输出正文时，可简要列出：

- `可补充材料`：支撑题目、问题、方法、进度或可行性所缺的信息。
- `可补充论点`：可以考虑但尚无充分依据的研究角度。
- `可展开论点`：现有材料已经支持、可继续深化的位置。

只列少量、具体、可操作的建议，不生成拟真研究设计、拟真数据或可被误当成事实的示例内容。用户要求只输出正文时，不附构造建议。

## 由小至大复核

- 段落：事实来源、计划状态和结论强度是否准确。
- 部分：是否完成对应任务，是否与中心问题和研究目标一致。
- 全文：题目、问题、目标、内容、方法、进度、条件和预期产出是否逐级对应；是否把拟开展事项写成既成事实，或新增未给出的研究成果。
- 交付：是否遵守用户模板、只输出正文、只输出改后稿或只审不改等要求；是否残留过程说明、规则自述或未完成占位。
